**components/CardDeck.tsx**

Horizontally scrollable row of selectable cards for voters.

Props:
```typescript
interface CardDeckProps {
  roomId: string
  myParticipantId: string
  selectedValue: string | null  // the locally-tracked selected card value
  onVoted: (value: string) => void  // called after successful vote POST, used to update local selected state
}
```

Implementation:
- Outer wrapper: `flex gap-3 overflow-x-auto py-2 px-1` (scrollable row, locked decision)
- Map over FIBONACCI_DECK (imported from @/lib/constants):
  ```tsx
  {FIBONACCI_DECK.map((value) => (
    <button
      key={value}
      onClick={() => handleVote(value)}
      className={[
        'shrink-0 w-14 h-20 rounded-lg border-2 transition-all duration-150',
        'focus:outline-none focus:ring-2 focus:ring-indigo-500 focus:ring-offset-2',
        selectedValue === value
          ? 'border-indigo-500 shadow-lg scale-[1.05] bg-indigo-50'
          : 'border-slate-200 bg-white hover:border-slate-400 hover:shadow-sm',
      ].join(' ')}
      aria-label={`Vote ${value}`}
      aria-pressed={selectedValue === value}
    >
      <span className="text-xl font-bold text-slate-800">{value}</span>
    </button>
  ))}
  ```
- `handleVote(value: string)`:
  - POST /api/rooms/${roomId}/vote with `{ value }`
  - On success: call onVoted(value) (parent updates selectedValue state)
  - On error: silent catch — vote display corrects on next refreshRoom cycle
- Touch target: `w-14 h-20` = 56x80px — exceeds 44x44px minimum (touch-target-size rule)
- Selected state uses border + shadow + scale (not color fill alone — color-not-only rule)
- aria-pressed reflects selected state (accessibility)
- No disabled cards are rendered — if the component is shown, all cards are always clickable (observer exclusion happens at parent level)
